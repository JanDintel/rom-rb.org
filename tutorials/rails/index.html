Tutorials » Rails

Welcome to the ROM on Rails tutorial.

This tutorial will help to explain some of the basic concepts behind ROM by building a simple Rails application to manage tasks. You will extend relations, define mappers, and use commands to modify data. It is primarily intended for those familiar with Rails who want to learn more about ROM.

  • Getting started - quickly initialize an application with a ROM-based template
  • Tasks index - display a list of tasks using a generated relation
  • Task relation - extend the tasks relation to order results
  • Task mapper - define a dedicated mapper for domain objects
  • Managing tasks - implement create/update/delete actions with commands
  • Validations - use validators

The following libraries are used throughout:

  • rom - main gem for the ROM project
  • rom-sql - database adapter based on Sequel
  • rom-rails - Rails integration for ROM

Of note is the rom-rails library, which is essentially a Railtie and some generators. The Railtie allows you to place relations, mappers, and commands in directories similar to how Rails places models, controllers, and mailers under the app directory. The integration makes ROM feel right at home in your Rails application.
